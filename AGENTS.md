# CourseIQ — Agent Guide

<!-- last-verified: 2026-03-12 -->

CourseIQ is a native iOS golf tracking app built with SwiftUI and SwiftData. Single-project Xcode workspace generated by XcodeGen.

## Quick Reference

| What | Where |

## Tech Stack

- **UI:** SwiftUI (iOS 17+)
- **Persistence:** SwiftData (SQLite-backed, `cloudKitDatabase: .none`)
- **Charts:** Swift Charts
- **Cloud:** CloudKit (lazy init, graceful degradation)
- **Project Gen:** XcodeGen
- **Swift:** 5.9+

## Commands

```bash
# Generate Xcode project (after changing project.yml or adding files)
xcodegen generate

# Build (command line)
xcodebuild -project GolfTracker.xcodeproj -scheme GolfTracker \
  -destination 'generic/platform=iOS Simulator' build

# Clean build
xcodebuild clean -project GolfTracker.xcodeproj -scheme GolfTracker
# or in Xcode: Product → Clean Build Folder (⇧⌘K)

# Clear derived data (nuclear option for build issues)
rm -rf ~/Library/Developer/Xcode/DerivedData/GolfTracker-*
```

## Conventions

### Code Style

- **No narration comments.** Don't add `// Import SwiftUI` or `// Define the view`. Comments only for non-obvious intent.
- **MARK comments** for section headers: `// MARK: - Section Name`
- **Private by default.** Views' helper functions and computed properties should be `private`.
- **Computed properties** over stored state when the value derives from other state.
- **`@Bindable`** for SwiftData model bindings in forms; `@Query` for fetching in views.

### SwiftUI Patterns

- Use `fullScreenCover(item:)` instead of `fullScreenCover(isPresented:)` when passing data — prevents nil-content white screens.
- Use `AppTheme` colors everywhere — never hardcode hex values.
- Apply `staggeredAppear(index:)` modifier to cards for entrance animations.
- Use `Haptics.light()` / `.medium()` / `.success()` / `.selection()` for feedback.
- Wrap related form fields in `cardSection(step:title:)` for consistent styling.

### Badge System

- Badges are defined in `BadgeType` enum in `Models/Badge.swift`.
- Detection runs in `BadgeManager.checkForBadges()`, called from `ActiveRoundView.submitCurrentHole()`.
- Session dedup uses a `Set<String>` keyed by `"badgeType-hole1,hole2"` — never rely on Date equality.
- Add new badge types: extend the enum, add detection method, call it from `checkForBadges()`.
- Badge images go in `Assets.xcassets/Badge<Name>.imageset/`.
- See `BADGES.md` for the full list of planned badges.

### Adding a New View

1. Create file in `Views/` directory.
2. Run `xcodegen generate` to update the Xcode project.
3. File is auto-discovered from the `GolfTracker/` source directory.

### Adding a New Model Property

1. Add property to the `@Model` class.
2. Add to `init()` parameters with a default value.
3. SwiftData handles lightweight migration automatically.
4. If migration fails at runtime, the app's `ModelContainer` setup in `GolfTrackerApp.swift` will attempt recovery (delete + recreate store).

## Do Nots

- **Don't** use `CKContainer(identifier:)` at init time — it crashes without iCloud entitlements. Use lazy/optional pattern (see `CloudKitManager.swift`).
- **Don't** compare `Date` objects for dedup after round-tripping through JSON/UserDefaults — floating-point precision loss breaks equality.
- **Don't** use `fullScreenCover(isPresented:)` with a separate optional state variable — use `fullScreenCover(item:)` instead.
- **Don't** put business logic in views — use `StatsEngine` for computation, `BadgeManager` for detection.
- **Don't** hardcode Haymaker course data in views — always go through `Course` model.
- **Don't** use `.foregroundStyle(.tertiary)` in contexts expecting `Color` — use `Color.gray.opacity(0.5)` instead.
